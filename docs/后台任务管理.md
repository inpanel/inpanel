## 重构完成总结

### 一、原 `BackendHandler` 逻辑与任务管理机制

`BackendHandler(RequestHandler)` 是一个 Tornado 请求处理器，处理 `/api/backend/{jobname}` 的 HTTP 请求，其核心任务管理机制如下：

**1. 状态管理**
- 使用类级变量 `jobs = {}` 和 `locks = {}` 维护全局任务状态
- `jobs` 字典存储任务信息，包含 `status`（running/finish/none）、`code`、`msg`、`data`
- `locks` 字典实现任务互斥锁，防止同一类型任务并发执行

**2. 任务生命周期方法**
- `_start_job(jobname)`: 启动任务，检查是否已在运行
- `_update_job(jobname, code, msg)`: 更新任务进度和状态
- `_finish_job(jobname, code, msg, data)`: 完成任务
- `_get_job(jobname)`: 获取任务状态
- `_lock_job(lockname)` / `_unlock_job(lockname)`: 任务互斥锁

**3. 异步任务调度**
- `_call(callback)`: 通过 `tornado.ioloop.IOLoop.instance().add_callback()` 将异步任务提交到事件循环
- POST 请求接收参数后，路由到对应的 `async def` 方法执行

### 二、重构方案

**1. 创建 `mod/task.py` - TaskManager 类**
- 包含完整的任务生命周期管理方法（`_lock_job`, `_unlock_job`, `_start_job`, `_update_job`, `_get_job`, `_finish_job`, `_call`）
- 通过构造函数注入 `settings` 和 `config` 依赖
- 迁移了所有异步任务方法（约 35+ 个），包括：
  - 系统管理：`update`, `service`, `datetime`, `ntpdate`, `swapon`, `mount`, `format`
  - 包管理：`yum_repolist`, `yum_installrepo`, `yum_info`, `yum_install`, `yum_uninstall`, `yum_update`, `yum_ext_info`
  - 文件操作：`move`, `copy`, `remove`, `compress`, `decompress`, `chown`, `chmod`, `wget`
  - MySQL 管理：`mysql_fupdatepwd`, `mysql_databases`, `mysql_dbinfo`, `mysql_users`, `mysql_rename`, `mysql_create`, `mysql_export`, `mysql_drop`, `mysql_createuser`, `mysql_userprivs`, `mysql_updateuserprivs`, `mysql_setuserpassword`, `mysql_dropuser`
  - SSH 管理：`ssh_genkey`, `ssh_chpasswd`
  - 远程管理：`inpanel_install`, `inpanel_uninstall`, `inpanel_config`
  - FTP 传输：`uploadtoftp`

**2. 精简 `web.py` - BackendHandler 类**
- 删除了所有任务管理方法（`_lock_job`, `_unlock_job`, `_start_job`, `_update_job`, `_get_job`, `_finish_job`, `_call`）
- 删除了所有异步任务方法（约 1300+ 行代码）
- 添加 `initialize()` 方法创建 `TaskManager` 实例
- 修改 `get()` 方法委托给 `self.task_manager._get_job()`
- 修改 `post()` 方法将所有任务调用委托给 `self.task_manager._call()` 和对应的任务方法

### 三、架构优化效果

| 优化前 | 优化后 |
|--------|--------|
| `web.py` 约 2400 行，职责混合 | `web.py` 约 990 行，专注 HTTP 请求处理 |
| 任务逻辑与请求处理耦合 | 任务逻辑独立到 `mod/task.py` |
| 难以单元测试 | `TaskManager` 可独立测试 |
| 修改任务逻辑需改动请求处理器 | 任务逻辑独立维护 |

```python
class TaskManager:
    jobs = {}
    locks = {}

    def __init__(self, settings, config):
        self.settings = settings
        self.config = config

    def _start_job(self, jobname):
        if jobname in self.jobs and self.jobs[jobname]['status'] == 'running':
            return False
        self.jobs[jobname] = {'status': 'running', 'msg': ''}
        return True

    # ... 其他生命周期方法

    async def update(self):
        if not self._start_job('update'):
            return
        # 执行升级逻辑...
        self._finish_job('update', code, msg)
```
